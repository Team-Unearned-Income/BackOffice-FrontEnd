# Knock-in BO — 공통 디스크립션

> 백오피스 전체 화면에 공통 적용되는 UX 정책 및 설계 기준

---

## 1. 접근 및 인증 `로그인`

1. **소셜 로그인**
   앱과 동일한 카카오/애플 소셜 로그인 사용. 별도 ID/PW 없음. 로그인 후 서버에서 `role` 확인 → `admin`이면 대시보드 이동, `user`이면 권한 없음 에러 노출.

2. **권한 구분**
   - `role = admin` : BO 접근 허용.
   - `role = user` : 접근 거부.
   - admin 권한은 **BO 회원 관리 > 상세 패널에서만** 부여/회수 가능. 본인 계정 셀프 권한 회수 불가.

3. **세션 관리**
   로그인 세션 만료 시 로그인 페이지로 자동 리다이렉트. 미로그인 상태로 BO URL 직접 접근 시 로그인 페이지로 이동.

---

## 2. 목록 화면 공통 `테이블`

1. **구성**
   검색/필터 영역 + 테이블 + 페이지네이션 구조.

2. **페이지네이션**
   모든 목록 페이지에 적용. 페이지당 노출 건수 및 UI는 **디자이너 재량**. 검색/필터 적용 시 1페이지로 초기화.

3. **처리 원칙**
   모든 처리(정지·비공개·삭제·권한 부여 등)는 **상세 보기 진입 후 상세 패널에서만** 수행. 목록에서 직접 처리 불가. 목록 액션 컬럼에는 "상세 보기" 버튼만 노출.

---

## 3. 상세 패널 공통 `패널`

1. **처리 후 상태**
   처리 완료 시 패널 내 상태값 즉시 업데이트. 모달에서 처리 확인 후 모달 닫힘 → 패널 상태 갱신. 처리자·일시 자동 기록.

---

## 4. 처리 확인 모달 공통 `모달`

1. **노출 조건**
   비가역적 처리(정지·삭제·비공개·권한 부여 등) 시 반드시 확인 모달 노출. 재노출·정지 해제처럼 간단한 처리는 사유 입력 없이 확인 버튼만.

2. **사유 입력**
   정지·비공개·삭제·반려: 사유 입력 **필수**. 미입력 시 처리 버튼 비활성화. 입력 사유는 내부 로그 저장 + 대상자 알림에 포함.

3. **버튼 구성**
   취소(좌) + 처리 버튼(우).
   - **취소**: 모달 닫힘, 처리 없음.
   - **처리**: 즉시 실행 + 처리자·일시 자동 기록.

---

## 5. 유저 알림 발송 기준 `알림`

### 처리 유형별 알림

| 처리 유형 | 발송 여부 | 알림 내용 |
| --- | --- | --- |
| 회원 정지 | 발송 | 앱 로그인 시 정지 안내 모달 노출 (사유 포함) |
| 회원 정지 해제 | 발송 | 고정 메시지: "계정 정지가 해제되었습니다. 서비스를 정상적으로 이용하실 수 있습니다." |
| 게시글 비공개 | 발송 | 푸시 알림: "게시글이 비공개 처리되었습니다. 사유: {사유}" |
| 게시글 재노출 | 발송 | 고정 메시지: "비공개 처리된 게시글이 다시 노출됩니다." |
| 게시글 삭제 | 발송 | 푸시 알림: "게시글이 삭제 처리되었습니다. 사유: {사유}" |
| 신원 인증 승인 | 발송 | 고정 메시지: "신원 인증이 완료되었습니다." |
| 신원 인증 반려 | 발송 | 앱 내 알림: "신원 인증이 반려되었습니다. 사유: {사유}" |
| 1:1 문의 답변 | 발송 | 고정 메시지: 고객센터 내 문의 내역에 답변 노출 + 앱 알림 |
| admin 권한 부여/회수 | 미발송 | — |

> **고정 메시지**: 관리자 사유 입력 없이 시스템에서 자동 발송하는 문구.
> **사유 포함**: 관리자가 모달에서 입력한 사유가 알림에 그대로 포함.

## 6. 처리 기한 기준 `운영`

### 메뉴별 처리 기한

| 메뉴 | 처리 기한 | 비고 |
| --- | --- | --- |
| 신고 관리 | 7 영업일 이내 | 접수 후 운영팀 알림 발송 |
| 신원 인증 관리 | 요청일로부터 3일 이내 | D-N 표시, D-0 초과 시 강조 (디자이너 재량) |

## 7. 데이터 처리 원칙 `개발`

1. **Soft Delete**
   - 탈퇴 회원: `is_deleted=true`, `deleted_at` 저장 → 3일 후 배치 완전 삭제.
   - 생활패턴 항목 삭제: `is_deleted=true`, 유저 응답 데이터(`UserLifePattern`) 보존, 앱 비노출.

2. **처리 로그**
   모든 처리(정지·비공개·삭제·권한 부여·반려 등) 시 **처리자 ID + 처리 일시 자동 기록**.

3. **즉시 반영**
   처리 완료 시 앱/BO 즉시 반영. 생활패턴 항목 변경 시 캐시 무효화 필요.
